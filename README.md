# PR Reviewer Service

## Запуск

```bash
docker-compose up
```
Проблемы и решения

1. В API нет отдельного эндпоинта для создания пользователей
В моем проекте пользователи создаются только через POST /team/add при создании команды

2. В спецификации нет кодов ошибок для случаев когда пользователи уже существуют
Я добавил код для такого случая: USER_EXISTS - пользователь уже существует

3. Не указано точное правило сколько ревьюверов назначать
Я назначал случайное количество от 0 до 2 активных ревьюверов из команды автора

4. Нет специального кода для внутренних ошибок сервера
Добавил свой код NO_PREDICTED для непредвиденных ошибок

5. В выводе всех PR нужно выводить те, которые необходимы в ревью или все
Я сделал разные эндпоинты для разных случаев:

/users/getReview - вывод всех pr

/users/getOpenReview - только те, которые нуждаются в ревью (открытые)
